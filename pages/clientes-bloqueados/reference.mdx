## Visão Geral

A **Lista de Clientes Bloqueados** é um mecanismo de prevenção de fraudes. Clientes bloqueados têm suas transações recusadas automaticamente em tempo real durante a criação de cobranças e assinaturas.

---

## Tipos de bloqueio

| Tipo    | Descrição                                        |
|---------|--------------------------------------------------|
| `email` | Bloqueia o cliente pelo endereço de e-mail       |
| `cpf`   | Bloqueia o cliente pelo número do CPF            |

---

## Campos retornados

| Campo           | Tipo     | Descrição                               |
|-----------------|----------|-----------------------------------------|
| `id`            | UUID     | Identificador único do registro         |
| `account_code`  | string   | Conta que realizou o bloqueio           |
| `email`         | string   | E-mail bloqueado (se tipo = email)      |
| `document`      | string   | CPF formatado (se tipo = cpf)           |
| `type_document` | string   | Tipo do documento (CPF, CNPJ, etc.)     |
| `type`          | string   | Tipo de bloqueio (Manual, Automatic)    |
| `created_at`    | datetime | Data/hora do bloqueio                   |

---

## Listagem com busca e múltiplas contas

```http
GET /v1/blocked-clients?accounts[]=acc_abc&accounts[]=acc_xyz&search=fraude@email.com&per_page=20
```

A listagem suporta filtro por múltiplas contas (`accounts[]`) e busca por e-mail ou CPF.

---

## Bloqueio em lote

```json Bloquear múltiplos e-mails
{
  "type": "email",
  "data": [
    "spam1@exemplo.com.br",
    "spam2@exemplo.com.br",
    "fraude@exemplo.com.br"
  ]
}
```

```json Bloquear múltiplos CPFs
{
  "type": "cpf",
  "data": [
    "123.456.789-09",
    "987.654.321-00"
  ]
}
```

<Tip>
  Clientes já bloqueados na lista são ignorados silenciosamente — não há erro ao tentar bloquear um cliente duplicado.
</Tip>

<Warning>
  O desbloqueio é feito pelos **IDs (UUIDs)** dos registros, não pelos e-mails ou CPFs.
  Obtenha os IDs via `GET /v1/blocked-clients`.
</Warning>
