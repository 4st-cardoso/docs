## Visão Geral

As **Faturas** são os pedidos (orders) gerados automaticamente a cada ciclo de cobrança de uma assinatura. Quando uma fatura não é paga, ela pode ser quitada manualmente via a rota de pagamento em atraso.

---

## Status de uma Fatura

As faturas utilizam o mesmo enum `StatusOrder` dos pedidos:

| Status             | Descrição                              |
|--------------------|----------------------------------------|
| `Pending`          | Aguardando pagamento                   |
| `Paid`             | Fatura paga com sucesso                |
| `Fail`             | Falha na tentativa de cobrança         |
| `Challenged`       | Aguardando confirmação do cliente      |
| `Canceled`         | Fatura cancelada                       |
| `ChargeBack`       | Estorno                                |
| `Reversed`         | Revertida                              |
| `PartialReversed`  | Parcialmente revertida                 |
| `Reproved`         | Reprovada                              |
| `New`              | Nova (recém-criada)                    |
| `Unprocessed`      | Não processada                         |

---

## Pagamento de fatura em atraso

Faturas com status `Pending` ou `Fail` podem ser pagas via `POST /v1/invoices/{invoice_code}/pay-overdue`. A assinatura precisa estar ativa para permitir o pagamento.

### Métodos aceitos

| Método       | Objeto `card`  |
|--------------|----------------|
| `CreditCard` | Obrigatório    |
| `DebitCard`  | Não necessário |
| `PIX`        | Não necessário |
| `Boleto`     | Não necessário |
| `PayPal`     | Não necessário |

```json Pagar fatura em atraso com cartão
{
  "method": "CreditCard",
  "card": {
    "name": "CARLOS FERREIRA",
    "number": "4111111111111111",
    "month": "12",
    "year": "28",
    "security_code": "321"
  }
}
```

```json Pagar fatura em atraso com PIX
{
  "method": "PIX"
}
```

<Info>
  O código da fatura (`invoice_code`) pode ser obtido via `GET /v1/subscriptions/{code}/invoices`.
  Este código corresponde ao campo `code` do pedido (order) da assinatura.
</Info>

<Note>
  A resposta de sucesso retorna diretamente os dados da cobrança criada, **sem** o envelope padrão `APIReturnUtil`.
  Todos os valores monetários (`amount`) são retornados em **centavos**.
</Note>
