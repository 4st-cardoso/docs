## Visão Geral

Os **Webhooks** são endpoints públicos que recebem notificações automáticas dos gateways de pagamento quando eventos ocorrem (ex: pagamento confirmado, estorno processado).

---

## Gateways suportados

| Gateway    | Endpoint                  |
|------------|---------------------------|
| Pagar.me   | `POST /webhook/pagarme`   |
| E-Rede     | `POST /webhook/erede`     |
| PayPal     | `POST /webhook/paypal`    |
| Genial     | `POST /webhook/genial`    |

---

## Configuração nos gateways

Configure a URL de webhook em cada gateway de pagamento com a URL completa do endpoint:

```
https://sua-api.com/api/webhook/pagarme
https://sua-api.com/api/webhook/erede
https://sua-api.com/api/webhook/paypal
https://sua-api.com/api/webhook/genial
```

---

## Eventos processados

| Evento               | Descrição                                  |
|---------------------|--------------------------------------------|
| `charge.paid`        | Pagamento confirmado → pedido aprovado     |
| `charge.failed`      | Pagamento falhou → pedido recusado         |
| `charge.refunded`    | Estorno processado → pedido estornado      |
| `subscription.paid`  | Fatura de assinatura paga                  |

<Info>
  Os endpoints de webhook não requerem autenticação Bearer Token. O payload é processado
  internamente e a assinatura do gateway é verificada conforme configurado em cada integração.
</Info>

<Warning>
  Não exponha os endpoints de webhook para uso manual. Eles são exclusivos para uso pelos gateways de pagamento.
</Warning>
